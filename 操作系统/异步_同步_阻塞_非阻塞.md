# 1. 同步与异步
同步异步是两者之间的状态

同步：二者步调一致。通常面试（一对一）是同步的。

异步：步调不一致。通常考试是异步的（出题的不用等做题的，做题的也不用管出题的）
# 2. 阻塞与非阻塞
阻塞：调用者必须等待到结果，才能结束等待。现场面试等结果一般是阻塞的（几个候选人当中选一个）。

非阻塞：调用者不需要等待，结果可能有也可能没有。线上面试等结果通常是非阻塞的（应该都懂，随缘结果）。
# 3. 同步i/o与异步i/o，阻塞i/o与非阻塞i/o
同步i/o：请求会阻塞，i/o完成结束阻塞。（读写数据要阻塞）

异步i/o：请求不会阻塞，i/o完成有结果。

阻塞i/o：必须等结果，等到结果才结束阻塞。

非阻塞i/o：资源暂时不可用（比如网络i/o读一个正在传的数据）会直接返回，可用时跟阻塞一样。

同步阻塞i/o：进程等待资源可用（同步，资源不可用时等待）并获取资源（这里阻塞）。

同步非阻塞i/o：进程等待资源可用（同步，资源不可用时不会等待）并获取资源（这里阻塞）。

异步非阻塞i/o：合理吗，正常异步不会阻塞。

异步阻塞i/o：呃呃，存在吗？如果进程调用异步i/o接口再等待i/o结果，实际上进程在做同步（等资源OK，也就是进程等函数i/o完成），用异步i/o接口实现同步操作。
## 4. 实际上还存在一种信号驱动i/o(SIGIO)
### SIGIO不是异步i/o。
进程通知内核当某个时间发生时用信号通知进程，进程之后i/o。
